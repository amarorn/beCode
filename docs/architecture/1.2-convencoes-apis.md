## 1. APIs síncronas

### 1.1 Estilo

- **REST + JSON** como padrão para APIs externas (SPA ↔ control plane).
- **gRPC** para comunicação interna entre serviços quando latência ou streaming justificam.
- **WebSocket** apenas para terminal e streams de IA.

### 1.2 Naming

- Recursos no **plural, kebab-case** (`/workspaces`, `/ai/completions`, `/extensions`).
- Ações fora de CRUD em **sub-recurso verbal** (`POST /workspaces/{id}/pause`).
- Campos JSON em **snake_case** (`tenant_id`, `created_at`).
- Enums em **UPPER_SNAKE** (`ALLOW`, `REQUIRE_APPROVAL`).

### 1.3 Versionamento

- Prefixo de versão major no path: `/v1/...`.
- Mudanças breaking ⇒ nova major.
- Mudanças aditivas compatíveis ⇒ não bump.

### 1.4 Identificadores

- UUIDv7 (sortable por tempo) para todos os IDs de recursos.
- `tenant_id` sempre no path ou no JWT; nunca só no corpo.

### 1.5 Erros

Formato alinhado a RFC 7807 (Problem Details):

```json
{
  "type": "https://errors.ide.corp/policy/denied",
  "title": "Policy denied",
  "status": 403,
  "detail": "command 'gcloud iam' is blocked for role=dev",
  "trace_id": "0af7651916cd43dd8448eb211c80319c",
  "policy_id": "terminal-default",
  "policy_version": 17
}
```

### 1.6 Paginação

- Cursor-based: `?limit=50&cursor=<opaque>`; resposta inclui `next_cursor`.
- Offset permitido apenas em endpoints admin de baixa cardinalidade.

### 1.7 Autenticação

- Cookie `HttpOnly Secure SameSite=Lax` transporta sessão entre SPA e API Gateway.
- Serviços internos usam **mTLS + token JWT curto** (≤5 min) assinado pelo Identity.
- Toda request carrega `tenant_id`, `user_id`, `squad_id`, `roles` no JWT.

### 1.8 Rate limit

- Header `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset` (epoch).
- Limit hierárquico: tenant → squad → user. AI Gateway tem limites específicos por modelo.

## 2. Eventos de domínio (Pub/Sub)

### 2.1 Envelope comum

```json
{
  "event_id": "uuidv7",
  "event_type": "workspace.ready",
  "event_version": 1,
  "occurred_at": "2026-04-18T12:34:56.789Z",
  "tenant_id": "t_...",
  "squad_id": "s_...",
  "actor": { "user_id": "u_...", "service": null },
  "trace_id": "0af7651916cd43dd8448eb211c80319c",
  "data": { /* payload específico do evento */ },
  "schema_ref": "pubsub://schemas/workspace/ready/v1"
}
```

### 2.2 Nomenclatura

- `<contexto>.<fato>` em snake_case: `workspace.ready`, `ai.completed`, `policy.violation`, `extension.approved`.
- Eventos são **fatos no passado** (sempre particípio).

### 2.3 Tópicos

- Um tópico por contexto (`workspace-events`, `ai-events`, `policy-events`, `data-events`, `identity-events`, `extension-events`).
- Consumidor de auditoria assina **todos**.

### 2.4 Schema evolution

- Schemas versionados no Schema Registry corporativo (GCP Pub/Sub Schemas ou equivalente).
- **Only-additive** em minor; breaking bumpa `event_version`.

### 2.5 Idempotência

- Todo consumidor deve ser idempotente por `event_id`.
- Produção usa `ordering_key = tenant_id` quando ordenação importa.

## 3. Telemetria (OpenTelemetry)

### 3.1 Tracing

- Todo serviço propaga W3C `traceparent`.
- Span por borda (HTTP in/out, gRPC, DB, cache, chamada a LLM, publicação de evento).
- Atributos obrigatórios: `tenant.id`, `squad.id`, `user.id`, `policy.id` (quando aplicável), `ai.provider`, `ai.model`.

### 3.2 Logs

- JSON estruturado; nível `INFO` default em prod, `DEBUG` habilitável por tenant via feature flag.
- Proibido logar: prompts brutos, conteúdo de queries do usuário, valores de secrets.
- Campos obrigatórios: `timestamp`, `level`, `service`, `trace_id`, `span_id`, `tenant_id`, `message`.

### 3.3 Métricas

Nomenclatura `<service>_<objeto>_<unidade>`:

- `api_request_duration_seconds` (histogram, labels: `service`, `route`, `method`, `status`).
- `policy_evaluations_total` (counter, labels: `decision`, `policy_id`).
- `ai_tokens_total` (counter, labels: `provider`, `model`, `direction=in|out`, `tenant`, `squad`).
- `ai_cost_micros_total` (counter).
- `workspace_provisioning_duration_seconds`.

### 3.4 Auditoria

Distinta de logs. Eventos auditáveis obrigatórios:

- `auth.login.*`, `auth.logout`
- `role.assigned`, `role.revoked`
- `policy.violation`, `approval.granted`, `approval.denied`
- `workspace.ready`, `workspace.terminated`
- `ai.requested` (somente hash do prompt), `ai.denied`, `ai.completed` (custo)
- `extension.submitted`, `extension.approved`, `extension.revoked`
- `secret.accessed` (quem/quando/qual escopo; nunca o valor)
- `data.query.executed` (hash + custo), `data.query.blocked_by_cost`

Retenção: 7 anos. Storage: GCS Bucket Lock + sink em BigQuery (task 11.2).

## 4. Segurança transversal

- **TLS** obrigatório em toda borda (inclusive intra-cluster quando possível via mTLS).
- **Secrets** nunca em variáveis de ambiente persistidas; obter via Secret Manager just-in-time.
- **Masking** de PII/secret em logs e em prompts de IA (pré-envio).
- **Feature flags** via serviço dedicado (out-of-scope de 1.2; referência futura).

## 5. Documentação obrigatória por serviço

Todo serviço mantém:

- `openapi.yaml` (REST) ou `.proto` (gRPC).
- `events.md` com tabela `{event_type, version, produtor, consumidores, schema}`.
- `runbook.md` (SRE) e `threat-model.md` (Security).
- Dashboards e alertas versionados em IaC (task 1.3).
