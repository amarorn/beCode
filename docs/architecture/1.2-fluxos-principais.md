## 1. Login corporativo (OIDC)

```mermaid
sequenceDiagram
    autonumber
    actor U as Usuário (browser)
    participant SPA as Web IDE Shell
    participant APIGW as API Gateway
    participant ID as Identity Service
    participant IdP as IdP corporativo
    participant R as Redis

    U->>SPA: Acessa ide.corp
    SPA->>APIGW: GET /auth/start
    APIGW->>ID: /auth/start (PKCE)
    ID-->>SPA: 302 IdP (state, code_challenge)
    U->>IdP: Autentica (MFA)
    IdP-->>SPA: 302 /auth/callback?code
    SPA->>APIGW: /auth/callback?code
    APIGW->>ID: troca code por tokens
    ID->>IdP: POST /token (PKCE verifier)
    IdP-->>ID: id_token + access_token
    ID->>ID: Mapeia claims → tenant/squad/roles
    ID->>R: cria sessão (httpOnly cookie)
    ID-->>SPA: 302 /workbench
```

**Pontos de observabilidade:** `auth.callback.latency`, `auth.failure.rate`, `claim_mapping.errors`.
**Auditoria:** `login.succeeded`, `login.failed`.

## 2. Provisionamento de Workspace

```mermaid
sequenceDiagram
    autonumber
    actor U as Dani
    participant SPA
    participant APIGW
    participant WM as Workspace Manager
    participant Q as Quota Svc
    participant T as Template Svc
    participant K8s as K8s (execution)
    participant Pod as Workspace Pod
    participant Bus as Pub/Sub

    U->>SPA: Escolhe template "dbt-bq" e clica "Iniciar"
    SPA->>APIGW: POST /workspaces
    APIGW->>WM: autenticado
    WM->>Q: valida cota do squad
    Q-->>WM: OK
    WM->>T: resolve template → imagem@digest
    T-->>WM: digest, quotas default
    WM->>K8s: apply Pod + NetworkPolicy + ServiceAccount
    K8s-->>Pod: cria
    Pod-->>K8s: ready
    K8s-->>WM: Ready
    WM->>Bus: publish workspace.ready
    WM-->>SPA: 201 {workspace_id, url}
    SPA-->>U: UI conecta via Terminal Proxy
```

**SLO:** p95 < 3 min (conforme task 1.1 §5.2).
**Auditoria:** `workspace.ready`, `workspace.terminated`.

## 3. Execução de comando no terminal (policy inline)

```mermaid
sequenceDiagram
    autonumber
    actor U as Dani
    participant SPA
    participant TP as Terminal Proxy
    participant PE as Policy Engine
    participant Pod as Workspace Pod
    participant AU as Audit

    U->>SPA: digita "gcloud iam ..."
    SPA->>TP: WS msg {line}
    TP->>TP: tokenize + hash
    TP->>PE: POST /evaluate (subject, action=terminal.exec, resource)
    PE-->>TP: DENY (reason="gcloud iam bloqueado p/ Dev")
    TP-->>SPA: WS msg {denied, reason}
    TP->>AU: event policy.violation
```

**Regra:** nenhum byte atinge o PTY do pod antes da decisão ALLOW.
**Fallback:** se Policy Engine indisponível > 500ms → **DENY** (fail-closed).

## 4. Prompt de IA governado

```mermaid
sequenceDiagram
    autonumber
    actor U as Dani
    participant SPA
    participant AIGW as AI Gateway
    participant PE as Policy Engine
    participant M as Masker
    participant C as Cache
    participant P as Provider LLM
    participant B as Bus
    participant AU as Audit

    U->>SPA: "Explique esta função" (+contexto do repo)
    SPA->>AIGW: POST /ai/completions
    AIGW->>PE: evaluate(ai.complete, tenant, squad)
    PE-->>AIGW: ALLOW (quota remaining: 72k tokens)
    AIGW->>M: mask(PII, secrets, PI-patterns)
    M-->>AIGW: prompt mascarado + map de tokens
    AIGW->>C: lookup semantic cache
    C-->>AIGW: miss
    AIGW->>P: complete(masked_prompt) [Anthropic | OpenAI | Vertex]
    P-->>AIGW: response
    AIGW->>AIGW: sanitize(response)
    AIGW->>B: publish ai.completed (tokens, cost, provider)
    AIGW->>AU: event ai.request (hash do prompt, não conteúdo)
    AIGW-->>SPA: resposta
```

**Invariante crítica:** o prompt enviado ao provedor é sempre o **mascarado**. O conteúdo original nunca deixa o control plane em claro.
**Cost:** evento `ai.completed` é a única fonte de verdade para chargeback.

## 5. Query BigQuery com guardrail de custo

```mermaid
sequenceDiagram
    autonumber
    actor U as Bea
    participant SPA
    participant D as Data Integrations
    participant PE as Policy Engine
    participant BQ as BigQuery
    participant AU as Audit

    U->>SPA: Executa query SQL
    SPA->>D: POST /queries/dry-run
    D->>BQ: jobs.insert (dryRun=true)
    BQ-->>D: bytesProcessed
    D->>PE: evaluate(query.execute, cost=estimated)
    alt custo ≤ threshold squad
      PE-->>D: ALLOW
      D->>BQ: jobs.insert (dryRun=false)
      BQ-->>D: jobId, resultados
      D-->>SPA: preview
    else custo > threshold
      PE-->>D: REQUIRE_APPROVAL
      D->>AU: event query.requires_approval
      D-->>SPA: bloqueado; workflow de aprovação
    end
```

**Regra:** nenhuma execução real sem dry-run prévio quando políticas de custo estão ativas.

## 8. Threat modeling inicial (STRIDE, resumo)

| Fluxo | Ameaça (STRIDE) | Mitigação |
|-------|-----------------|-----------|
| 1 (login) | **S** spoofing de sessão | PKCE + cookie `HttpOnly; Secure; SameSite=Lax`; TTL curto |
| 1 | **R** repúdio de ações | Evento `login.succeeded` imutável no Audit |
| 2 (workspace) | **E** elevação via spec K8s | Admission controller + PodSecurityStandard `restricted` |
| 2 | **I** vazamento via volume | Volumes efêmeros, sem PV compartilhado entre tenants |
| 3 (terminal) | **T** tampering no canal | TLS obrigatório; WS autenticado com token de sessão |
| 3 | **E** bypass de policy | Fail-closed; decisão síncrona antes do PTY write |
| 4 (IA) | **I** vazamento de PII | Masking pré-envio; redaction de logs; sem logging do prompt bruto |
| 4 | **D** DoS via custo | Rate limit hierárquico; quota por squad; circuit breaker por provider |
| 5 (BigQuery) | **D** custo descontrolado | Dry-run obrigatório + threshold por squad |
| 6 (extensão) | **T** supply chain | Pull por digest; verificação de assinatura (cosign); SBOM |

Threat modeling detalhado será registrado em `docs/security/threat-model-v0.md` como entregável complementar.
