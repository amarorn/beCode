**Task:** 1.1 — Definir visão do produto, personas e boundaries
**Versão:** 1.0.0
**Data:** 2026-04-18

> Baseline RICE × MoSCoW para as 13 épicas do backlog, ancorada nas personas e jornadas definidas em `1.1-visao-produto.md` e `1.1-mapa-jornadas.md`. Serve como input oficial para o refinamento das tasks e para a arquitetura (task 1.2).

---

## 1. Princípios de priorização

1. **Aderência a persona** — épico sem persona primária vinculada é rebaixado.
2. **Destrava jornada crítica (J1, J2, J4, J6)** — ganha prioridade *Must*.
3. **Governança é pré-requisito, não extra** — segurança, audit e policy entram no MVP.
4. **Diferencial competitivo > paridade** — paridade com VS Code é condição de entrada, diferencial (IA governada, policy) é o motor de adoção.
5. **Sem lock-in** — épicos que reforçam abstrações multi-LLM/multi-registry são valorizados.

## 4. Regras de descarte / repriorização

- Qualquer feature nova deve citar **persona + job** no DoR (regra SpecDrive).
- Features de "produtividade" sem persona primária vinculada → *Won't (MVP)*.
- Features que exijam *editor engine próprio* → bloqueadas até Fase 3+.
- Features dependentes de Databricks que adicionem >1 sprint de esforço → Fase 2.
